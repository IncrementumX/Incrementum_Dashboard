# raw/ — raw sources (Karpathy LLM Wiki pattern)

> Source: [karpathy/llm-wiki gist](https://gist.github.com/karpathy/442a6bf555914893e9891c11519de94f). Quotes below are LITERAL from Karpathy — do not translate or reinterpret.

## What this is (Karpathy, literal)

> *"Raw sources — your curated collection of source documents. Articles, papers, images, data files. These are immutable — the LLM reads from them but never modifies them."*

This is **layer 1 of 3** in the LLM Wiki pattern. The other two layers exist already in this repo:

- **The schema** — `IncrementumOS/governance/CLAUDE.md` ("a document that tells the LLM how the wiki is structured, what the conventions are, and what workflows to follow").
- **The wiki** — `IncrementumOS/wiki/` ("a directory of LLM-generated markdown files. Summaries, entity pages, concept pages, comparisons, an overview, a synthesis").

## Subfolders

```
raw/
├── articles/   # web articles, blog posts, Substack/Medium pieces (often via Obsidian Web Clipper)
├── papers/     # academic papers, SEC filings (10-K, 10-Q), IR decks, transcripts
└── data/       # spreadsheets, exports, screenshots, charts, raw datasets
```

## Naming convention

```
YYYY-MM-DD_<source>_<short-slug>.md
```

Examples:
- `articles/2026-04-28_bloomberg_micron-hbm-supply.md`
- `papers/2026-04-15_micron_10-Q_Q3-2026.pdf`
- `data/2026-04-20_screen_memory-companies.csv`

If date is unknown, use `undated_<source>_<short-slug>.<ext>`. Source rules from `governance/CLAUDE.md` apply (no Robinhood / StockTitan / Investing.com).

## Three core operations (Karpathy, literal)

> **Ingest** — *"You drop a new source into the raw collection and tell the LLM to process it... A single source might touch 10-15 wiki pages."*

> **Query** — *"You ask questions against the wiki. The LLM searches for relevant pages, reads them, and synthesizes an answer with citations."*

> **Lint** — *"Periodically, ask the LLM to health-check the wiki. Look for: contradictions between pages, stale claims... orphan pages with no inbound links."*

## Two special files

- **`../wiki/index.md`** — *"content-oriented... each page listed with a link, a one-line summary"*
- **`../wiki/log.md`** — *"append-only record of what happened and when"*

## Key principle

> *"The wiki is a persistent, compounding artifact."*

## What goes in `raw/` vs what does NOT

**Goes in:**
- Web articles (clipped via Obsidian Web Clipper).
- SEC filings, IR materials, earnings transcripts.
- Bloomberg captures (with `[Bloomberg, DD-MMM-YYYY]` tag in the filename or front-matter).
- Screenshots, charts, raw data exports.

**Does NOT go in:**
- Anything generated by an LLM — that belongs in `wiki/`.
- Personal journal / capital decisions / private notes — keep local, outside this repo.
- Sensitive documents (PII, confidential filings) — keep outside this repo.

## How to use

**Ingest:** drop a new source into the relevant subfolder; ask Claude (analista) to ingest it; Claude updates pages in `wiki/` and appends to `wiki/log.md`.

**Query:** ask Claude a question; Claude reads `wiki/index.md` and the relevant pages, synthesizes with citations to both `wiki/` and `raw/`.

**Lint:** periodically (e.g., every 2 weeks), ask Claude to lint the wiki — flag contradictions, stale claims, orphan pages.

Sources are immutable. If a fact in a `raw/` document is wrong or superseded, do **not** edit the source — instead, the wiki reflects the correction with a citation that says "supersedes raw/<file>".
