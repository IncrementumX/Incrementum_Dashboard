# teses/ — teses de portfólio (evolutivo → Atlas)

Cada arquivo aqui é uma tese viva: por que estamos comprados/vendidos em algo, o que precisa ser verdade, o que invalidaria, qual o catalisador, qual o sizing.

## Status

**Em construção.** Por ora, Atlas = manual operacional. Teses entram aqui primeiro; quando maduras, migram para o Atlas.

## Formato sugerido (não-vinculante)

```markdown
# <Ticker / Tema> — tese
- **Direção:** long / short / hedge
- **Sizing-alvo:** X% NAV
- **Tese central:** 2–3 parágrafos
- **O que precisa ser verdade:** lista
- **Catalisadores:** lista com timing
- **Invalidação / stop:** condição clara
- **Riscos não-óbvios:** lista
- **Última revisão:** YYYY-MM-DD
```

## Lifecycle

1. Hipótese → arquivo aqui em `teses/`.
2. Convicção sobe → posição entra → `state.md` reflete.
3. Tese matura, sustentada por trabalho → migra para Atlas.
4. Tese invalidada → marcar status, manter histórico (não deletar).
