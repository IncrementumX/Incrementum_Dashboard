# framework/ — critérios, processos, checklists

Aqui vivem os "como" práticos: o que checar antes de uma posição, como estruturar um short, como pensar em sizing, qual o processo de revisão. Mais operacional que `philosophy/`, mais estável que `teses/`.

## Status

**Em construção.** Vazio até Eduardo subir material ou até decisões repetidas justificarem formalização.

## Exemplos do que entra aqui (não-vinculante)

- Checklist de tese (mínimo de evidências antes de comprar)
- Critérios de short (qualidade do thesis, catalisador, custo de borrow, tail risk)
- Processo de sizing (como dimensionar conforme convicção × risco × correlação com book)
- Critérios de hedge (quando proteger via put vs. short index vs. reduzir bruto)
- Quando rotacionar
- Quando cortar

## Princípio

Framework é vivo. Quando uma regra não funcionar, atualizamos — não vinculamos a versões antigas. Cada arquivo tem data de última revisão.
