# Entry & Exit Checklist

> Synthesized from @incrementum-associate session, 2026-05-01.
> Primary audience: incrementum-analista and incrementum-associate agents.
> Status: v1 — approved by Eduardo on 2026-05-01.

## Entry Checklist

1. **Thesis**: is there a partial or full thesis? If partial, does an identifiable
   asymmetry exist? If neither, do not enter.

2. **Sizing**: conviction level maps to sizing-rules.md tier. If conviction is
   incomplete or valuation is stretched without compensating asymmetry, max
   entry is Seed (≤2%) or Small-asymmetric (≤5%). Urgency is not a sizing input.

3. **Macro context**: is the macro tape adverse to the position? If yes, does the
   asymmetry still hold? If not, wait or reduce initial size.

4. **Stop / invalidation**: what would falsify the thesis? Is it observable? For
   shorts, stop level must be defined before entry.

5. **Correlation**: does this position increase concentration in an existing theme?
   If yes, flag — does not block entry, but sizing must reflect combined exposure.

## Exit Checklist

### 1. Thesis Invalidated

Primary falsification condition has been met. Exit fully, promptly. This is not a
loss-cut — it is a logical conclusion. Do not average down after thesis breach.

### 2. Valuation Stretched / Target Reached

No explicit price target required. Exit is qualitative: risk/reward has deteriorated
to the point where capital is better deployed elsewhere. Partial exits are valid —
reduce to a size consistent with the new risk/reward, do not force binary decision.

### 3. Capital Rotation

A higher-conviction opportunity requires capital. Evaluate current position's
conviction tier vs. the new opportunity. Rotate from lower tier to higher tier,
not the reverse.

## General Rules

- Noise vs. signal distinction (per portfolio-risk-limits.md): price moves driven
  by beta or macro sentiment are not exit triggers. Thesis breach is.
- Partial exits are always available. Binary hold/sell is not the only option.
- Re-entry after exit is a fresh decision — prior cost basis is irrelevant.
